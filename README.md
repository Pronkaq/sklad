# MVP учёта поддонов ткани

Локальное веб-приложение для учёта движения поддонов между цехами, складом, обработкой, разбраковкой, ТО и лабораторией.

## Что умеет MVP

- Создание поддона цехом 10 / 3 / Новгород
- Генерация уникального номера поддона
- Генерация QR-кода
- Печать этикетки поддона
- Приёмка поддона в цех 5
- Передача поддона на обработку
- Завершение обработки с созданием новой этикетки
- Передача на разбраковку
- Разделение поддона по направлениям:
  - ТО
  - Склад
  - 7 лаборатория
  - Брак
  - Условно годен
- История движения каждого поддона
- Остатки по стадиям
- Выгрузка остатков в Excel

## Установка

Нужен Python 3.10+

```bash
cd pallet_mvp
python -m venv venv
```

### Windows

```bash
venv\Scripts\activate
pip install -r requirements.txt
python app.py
```

### Linux

```bash
source venv/bin/activate
pip install -r requirements.txt
python app.py
```

После запуска открыть:

```text
http://127.0.0.1:8000
```

Если нужно открыть с телефона в той же Wi-Fi сети, на ПК узнай IP:

```bash
ipconfig
```

Например, если IP ПК `192.168.1.50`, то с телефона открыть:

```text
http://192.168.1.50:8000
```

## Как печатать этикетки

В карточке поддона нажать "Этикетка". Откроется печатная страница. Дальше Ctrl+P.

На старте можно печатать на обычном принтере А4 и клеить этикетку на поддон.

## Логика ролей в версии v3

### Цеха 10, 3 и Новгород

Эти учётки нужны только для создания этикетки передачи в цех 5.

Они могут:
- создать поддон;
- выбрать/указать ассортимент;
- указать рулоны, метраж, дату наработки;
- распечатать A5-этикетку с QR;
- смотреть карточку поддона.

Они не могут:
- принимать поддон на склад цеха 5;
- передавать на обработку;
- завершать обработку;
- запускать разбраковку;
- разделять поддон после разбраковки;
- выгружать остатки в Excel.

### Цех 5

Учётки цеха 5 делают всю дальнейшую работу:
- приёмка на склад;
- передача на КТУ/ЛКТМ;
- завершение обработки;
- печать новой этикетки;
- разбраковка;
- распределение на ТО / склад / 7 лабораторию / брак / условно годен;
- Excel-выгрузка остатков.

## Версия v4: запуск на Python 3.14

В этой версии убрана зависимость `Pillow`, потому что на Python 3.14 под Windows она может пытаться собираться из исходников и падать.

QR-код теперь генерируется в SVG-формате:

```text
/pallet/<id>/qr.svg
```

Для браузера и печати это нормально: SVG отображается как обычная картинка.


## Версия v5: встроенный сканер QR

Добавлена страница:

```text
/scan
```

В интерфейсе появилась кнопка `Сканировать QR`.

Важно:
- встроенный сканер использует браузерный `BarcodeDetector`;
- лучше работает в Chrome/Edge на Android;
- если браузер не поддерживает сканер, можно открыть обычную камеру телефона — QR содержит ссылку на карточку поддона;
- также есть ручной ввод ID поддона.


## Версия v6: сканер через ZXing

Страница `/scan` теперь использует библиотеку ZXing через CDN:

```text
https://unpkg.com/@zxing/browser@latest
```

Это лучше, чем `BarcodeDetector`, потому что `BarcodeDetector` поддерживается не всеми браузерами.

Важно:
- на телефоне камера почти всегда требует HTTPS;
- если сайт открыт по `http://IP`, браузер может не дать доступ к камере;
- QR всё равно можно открыть обычной камерой телефона;
- ручной ввод ID поддона остаётся.

## Версия v10: исправление SQLite `database is locked`

Исправление:
- SQLite подключается с `timeout=30`;
- включается `PRAGMA busy_timeout`;
- включается WAL-режим;
- генерация номера поддона использует `BEGIN IMMEDIATE` и retry;
- для VPS всё равно рекомендуется Gunicorn с одним worker:
  `--workers 1`.
